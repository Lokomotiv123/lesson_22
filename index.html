let user;
let userName;
userName = "Вася";
user = userName;
console.log(user);

Варіант №1

let user = "Вася" let age = 36

Відповідь: написано не вірно. Відсутня кома між виразами, ключове слово let другий раз не потрібно, в кінці має бути крапка з комою.

Варіант №2

Відповідь: написано не вірно. Назва змінної записано не за правилами. Друге слово має бути з великої букви.

Варіант №3

const BLOCK_SIZE = 14 + 50;

Відповідь: написано не вірно.Оскільки це константа, значення якої присвоюється в процесі виконання програми, назва має бути написана за правилами для змінних.

Варіант №4

const BLOCK_HEIGHT = 100;

Відповідь: написано вірно.Для констант, значення яких відомі з самого початку, назва пишеться у верхньому регістрі та слова розділяються нижнім підкресленням.

Варіант №5

if (true) {
var size = 15;
}
console.log(size);

Відповідь: Написано вірно. Якщо змінну було об'явлено з допомогою ключогового слова var в блоці то з цією змінною можна працювати за межами блоку. В косоль виводиться цифра 15.

Домашка №3 JS (тип данних)

Варіант №1

let userAge = 36;
let userInfo = "Фрілансер ${userAge}";

Відповідь: не вірно. Для того, щоб додати до виразу змінну потрібно обгортати його в зворотні лапки ``.

Варіант №2

let userHeight = 145 / 0;
// Повернеться значення NaN
console.log(userHeight);

Відповідь: не вірно. Повернеться значення Infinity.

Варіант №3

let userName;
// Повернеться тип даних Null
console.log(typeof userName);

Відповідь: не вірно. Повернеться тип данних undefined.

Варіант №4

let userSize = "45" / "8";
//Повернеться тип данних number

Відповідь: вірно. Повернеться тип данних number.
